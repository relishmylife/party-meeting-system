## 📋 详细使用说明

### 1️⃣ 开发环境启动

**适用场景**: 本地开发、功能测试、快速演示

**启动命令**:
```bash
./quick-dev-start.sh
```

**特点**:
- ✅ 无需sudo权限
- ✅ 自动检查和安装依赖
- ✅ 智能处理端口冲突
- ✅ 启用热重载功能
- ✅ 快速启动 (~2分钟)

**访问方式**:
- 本地访问: http://localhost:5173
- 网络访问: http://[服务器IP]:5173

---

### 2️⃣ 生产环境启动

**适用场景**: 正式部署、学校集成、长期运行

**启动命令**:
```bash
sudo bash start-all-services.sh
```

**特点**:
- ✅ 完整的系统服务管理
- ✅ Nginx反向代理配置
- ✅ SSL证书准备
- ✅ 健康检查和监控
- ✅ 生产级别错误处理

**访问方式**:
- HTTP访问: http://localhost/
- HTTPS访问: https://party.imust.edu.cn/

---

### 3️⃣ 混合模式启动

**适用场景**: 开发测试 + 生产演示

**启动命令**:
```bash
sudo bash start-all-services.sh dev
```

**特点**:
- ✅ 启动开发服务器 (端口5173)
- ✅ 同时保持Nginx服务 (端口80)
- ✅ 双重访问地址支持
- ✅ 开发调试友好

**访问方式**:
- 开发地址: http://localhost:5173 (热重载)
- 生产地址: http://localhost/ (生产版本)

## ⚠️ 重要提醒

### 权限要求
- **开发脚本**: 无需sudo权限
- **启动脚本**: 需要sudo权限
- **停止脚本**: 需要sudo权限

### 环境要求
- **操作系统**: Linux (Ubuntu 20.04+ / CentOS 7+)
- **Node.js**: >= 16.0.0
- **内存**: 最低1GB，推荐2GB+
- **磁盘**: 最低5GB可用空间

### 网络配置
- **Supabase**: 需要访问外网
- **防火墙**: 需要开放80、443、5173端口
- **域名**: 解析到服务器IP地址

## 🔧 故障排除

### 常见问题快速解决

**问题1: 权限错误**
```bash
# 解决: 使用sudo运行
sudo bash start-all-services.sh
```

**问题2: 端口被占用**
```bash
# 解决: 脚本会自动处理或手动清理
sudo bash stop-all-services.sh --force
sudo bash start-all-services.sh
```

**问题3: 依赖安装失败**
```bash
# 解决: 清理缓存重新安装
rm -rf node_modules pnpm-lock.yaml
./quick-dev-start.sh
```

**问题4: Supabase连接失败**
```bash
# 解决: 检查网络连接
curl -I https://lfmpvxczahvcselayyho.supabase.co
```
