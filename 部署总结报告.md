## 🚀 部署方案详解

### 方案1: 独立域名部署 ⭐⭐⭐⭐⭐

**适用场景**: 希望系统完全独立运行，获得完整控制权

#### 优势
- ✅ 完全独立，风险可控
- ✅ 可以自定义域名和SSL证书
- ✅ 支持任意功能定制
- ✅ 不依赖现有系统稳定性
- ✅ 便于演示和推广

#### 部署步骤
1. **选择云服务商** (推荐阿里云/腾讯云)
2. **申请域名** (如: party-manage.com)
3. **购买云服务器** (1核2G起步)
4. **运行部署脚本**: `bash deploy.sh`
5. **配置SSL证书**: `bash ssl-certificate.sh`
6. **测试系统功能**

#### 成本估算
- 服务器: ¥120-300/月
- 域名: ¥50-80/年
- SSL证书: 免费 (Let's Encrypt)
- **总成本**: ¥1500-4000/年

### 方案2: 学校子域名部署 ⭐⭐⭐⭐

**适用场景**: 希望与学校系统保持一定关联，但维持独立性

#### 优势
- ✅ 使用学校域名，用户信任度高
- ✅ 便于学校管理和监控
- ✅ 可以申请学校资源支持
- ✅ 网络环境更稳定

#### 集成方式
```
内蒙古科技大学门户
    ├── 党建系统入口 (party.imust.edu.cn)
    └── 其他子系统...
```

#### 申请流程
1. 向学校网络中心申请子域名
2. 提供系统说明和技术方案
3. 配置DNS解析到云服务器
4. 部署SSL证书

### 方案3: 完全集成到学校门户 ⭐⭐⭐

**适用场景**: 希望与学校现有系统深度集成

#### 集成方式A: iframe嵌入
```html
<!-- 在学校门户中添加iframe -->
<iframe src="https://party.imust.edu.cn" width="100%" height="800px"></iframe>
```

#### 集成方式B: SSO单点登录
- 复用学校统一身份认证
- 用户无需额外登录
- 权限基于学校角色映射

#### 集成方式C: 数据同步
- 与学校用户数据库集成
- 自动同步组织架构信息
- 统一用户管理

## 🏫 与内蒙古科技大学集成的最佳实践

### 推荐方案: 学校子域名部署

**为什么推荐？**
1. **平衡性最佳**: 既保持独立性，又体现学校归属
2. **实施难度适中**: 不需要复杂的系统对接
3. **用户体验良好**: 域名权威，访问稳定
4. **管理便利**: 学校可以监控和管理

### 具体实施步骤

#### 第一阶段: 准备 (1周)
1. **向学校申请资源**
   - 子域名: `party.imust.edu.cn`
   - SSL证书配置
   - 网络访问权限

2. **技术准备**
   - 云服务器采购
   - 域名DNS配置
   - SSL证书申请

#### 第二阶段: 部署 (1周)
1. **服务器部署**
   ```bash
   # 上传系统文件
   scp -r dist/* user@server:/var/www/party-system/
   
   # 运行自动部署脚本
   ssh user@server "sudo bash deploy.sh"
   ```

2. **域名配置**
   - DNS A记录: party.imust.edu.cn → 服务器IP
   - SSL证书: 使用Let's Encrypt自动申请

#### 第三阶段: 集成 (1周)
1. **学校门户集成**
   ```html
   <!-- 在学校门户导航中添加 -->
   <li>
       <a href="https://party.imust.edu.cn" target="_blank">
           <i class="icon-users"></i> 党组织生活
       </a>
   </li>
   ```

2. **用户培训**
   - 系统使用培训
   - 管理员培训
   - 技术支持培训

### 集成配置说明

#### 关键配置文件
1. **`imust-integration.conf`** - 学校集成配置
2. **`.env.production`** - 生产环境配置
3. **`deploy.sh`** - 自动部署脚本

#### 重要配置项
```yaml
# 域名配置
DOMAIN: party.imust.edu.cn

# SSO集成
SSO_CONFIG:
  enabled: true
  issuer: https://cas.imust.edu.cn
  
# 权限映射
ROLE_MAPPING:
  super_admin: ["admin", "super_admin"]
  admin: ["party_secretary", "party_organizer"]
  normal_user: ["party_member"]
```
